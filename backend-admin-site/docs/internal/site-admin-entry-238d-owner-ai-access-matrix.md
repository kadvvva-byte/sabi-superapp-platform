# Sabi Site 238D — Owner / Sabi AI / Admin Access Matrix

INTERNAL ONLY — not public website content.

| Role | Target access | Boundary |
|---|---:|---|
| Owner | 100% | Final authority for all live actions and approvals. |
| Owner Sabi AI | 100% visibility, report-only execution | Monitors, analyzes, reports, escalates. Does not execute final actions independently. |
| Deputy | Up to 80% | Delegated operational control, no final financial/provider launch authority without Owner approval. |
| Staff | 10–50% | Function-scoped, least privilege, only assigned modules/screens. |
| Developer | Technical scope only | No financial/provider activation without Owner approval. |
| Accountant | Finance report scope | Reports and reconciliation, no final release without Owner approval. |

## Function-scoped access
Admin access must be function-scoped and least privilege. Taxi staff see Taxi responsibilities only. Stream staff see Stream responsibilities only. Finance staff see assigned finance/reporting areas only.

## Mail/Admin connection
Corporate mail state is verified inside Admin, not on the public site. Admin should show mailbox creation, DNS verification, inbound/outbound test status, DMARC monitoring, alerts, and Sabi AI daily risk report.
